## 필요한 작업 목록

### 1. 사업자 정보 (푸터 또는 별도 페이지)
- [ ] 상호명
- [ ] 대표자명
- [ ] 사업자등록번호
- [ ] 통신판매업 신고번호 (해당 시)
- [ ] 주소
- [ ] 고객센터 이메일/연락처

### 2. 이용약관 페이지 (`/terms`)
- [ ] 서비스 이용약관 전문
- [ ] 링크를 사이트 내 접근 가능한 위치에 노출 (보통 푸터)

### 3. 개인정보처리방침 페이지 (`/privacy`)
- [ ] 개인정보처리방침 전문
- [ ] 수집 항목, 목적, 보유기간, 제3자 제공 여부 등 포함
- [ ] 링크를 사이트 내 접근 가능한 위치에 노출 (보통 푸터)

### 4. 환불 정책 페이지 또는 섹션
- [ ] 크레딧 환불 조건 명시
- [ ] 이용약관 또는 별도 `/refund` 페이지에 포함 가능
- [ ] "환불", "취소" 키워드가 명확히 표시되어야 함

### 5. 상품 정보 노출
- [ ] 메인 또는 구매 페이지에 크레딧 상품 명시
- [ ] "1 Credit / 2,990원", "5 Credits / 8,990원" 등 상품명+가격 노출
- [ ] "구매", "결제", "서비스" 등 커머스 키워드 포함

## 구현 위치 제안

```
푸터 (모든 페이지 공통)
├── 이용약관 → /terms
├── 개인정보처리방침 → /privacy
├── 환불정책 → /refund (또는 이용약관 내 섹션)
└── 사업자 정보 (상호명, 사업자번호, 대표자, 주소, 연락처)

/buy 페이지
└── 크레딧 상품 목록 + 가격 명시 (이미 있을 수 있음, 키워드 확인 필요)
```

## 참고

- PG사 심사는 자동 크롤러로 키워드를 검색하는 방식
- "이용약관", "개인정보", "환불", "상품", "구매" 등의 키워드가 사이트 HTML에 존재해야 함
- 각 페이지는 로그인 없이 접근 가능해야 함 (크롤러가 비로그인 상태로 검사)
- 모바일 반응형 필수 (PG심사 기준)
